# v0.75.5

Despite recent focus on Kweave, there's been some useful additions to the PCGEx toolkit!

## New Nodes

### [Cluster : BFS Depth](../../node-library/clusters/analyze/cluster-bfs-depth.md)

Handy node that does a seed-based Depth-first search and writes down the shorted connection depth as an attribute; optionally computes the distance and can output seed index for easy partitioning.

<figure><img src="../../.gitbook/assets/image (329).png" alt=""><figcaption></figcaption></figure>

### [Dynamic Mesh to Cluster](v0.75.5.md#dynamic-mesh-to-cluster)

Title. It's the dynamic mesh friendly version of [mesh-to-clusters.md](../../node-library/clusters/generate/mesh-to-clusters.md "mention") — only difference is that one doesn't use target points to copy the output clusters to, it matches the point geometry as-is.

<figure><img src="../../.gitbook/assets/image (330).png" alt=""><figcaption></figcaption></figure>

### [Cull On Empty](../../node-library/utilities/cull-on-empty.md)

A very simple node with a special culling output pin. Helps streamlining graph culling if the inputs contains no valid or only empty data. _Especially useful early in partitioned graphs that rely on external data that may not exist in a given partition._

Support dynamic types, but is only meaningful for Attribute Sets, Polylines (Splines, Polygons etc), and Points.&#x20;

> Anything else that "exists" is considered valid.

## New Filters

{% hint style="info" %}
## String comparison modes across PCGEx (tags, filters, compare etc) now support wildcards.

You can use `*` or `?` as part of a string to a multi-contain-in-order match; i.e `*B*D*` wll match `ABCD` or `EhBoyWhyYouDoThis`.

_Note that this has been supported by vanilla forever, I just never realized it wasn't a simple Contains check._
{% endhint %}

Also, there's a handful of new filters!

### [Volume Inclusion Filter](../../node-library/filters/point-filters/spatial/filter-inclusion-volume.md)

A simple filter that lets you check a point relationship to volume actors / volume data. It's similar to the likes of [filter-inclusion-bounds.md](../../node-library/filters/point-filters/spatial/filter-inclusion-bounds.md "mention")except that it only support a radius, not box check.

<figure><img src="../../.gitbook/assets/image (333).png" alt=""><figcaption></figcaption></figure>

### [Primitive Overlap Filter](../../node-library/filters/point-filters/spatial/filter-primitive-overlap.md)

A simple filter that lets you check, you guessed it, point against primitive. This is straight using available collisions from a list of primitive so it's a rather precise way to check if something is inside or outside a bunch of meshes.&#x20;

> Since it's physic based this is an expensive check, on top of being "just" a binary check. Check is octree-backed so it's still _very_ efficient when points are small and well distributed : physic is skipped if there's no box/box intersection.

<figure><img src="../../.gitbook/assets/image (332).png" alt=""><figcaption></figcaption></figure>

### Regex + Edge Endpoints Regex

Similar to a string compare, but lets you use a Regex match pattern instead.

## Misc

#### CI/CD PCHs

Added a PCGEX\_NO\_PCH environment variable support in addition to the .noPCH file; more handy for CI/CD pipelines.&#x20;

> Just add a PCGEX\_NO\_PCH = "1" and you don't have to care about PCH issues anymore.

#### Tweaks

* Exposed some more prioritization options for [path-stitch.md](../../node-library/paths/modify/path-stitch.md "mention") _(more control over which path can/should stitch first based on various criterias like distance, alignment etc)_
* Revamped PCG Data Asset Collection generation flow
  * Support level reference & auto-generate embedded PCG Data Asset straight into the collection.
  * Uses custom BP-friendly exporter that has automatic filtering of things and custom bound computations (see [bounds-evaluator.md](../../node-library/staging/collections/helpers/bounds-evaluator.md "mention") and [level-data-exporter-default.md](../../node-library/staging/collections/pcg-data-asset-collection/level-data-exporter-default.md "mention"))

* Collection editor & property override multi editing fixes
* DataMatching rare crash with [copy-to-points.md](../../node-library/transform/generate/copy-to-points.md "mention")
* Spawned actor leftovers when using [staging-spawn-actors.md](../../node-library/staging/staging-spawn-actors.md "mention")
* Fixed a hard crash with sequential growth when using [cluster-flood-fill](../../node-library/pathfinding/cluster-flood-fill/ "mention")
